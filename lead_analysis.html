⚠️ 부재 41% — 콜 운영의 구조적 병목

DB 10건 중 4건이 부재로 묶임. 광고비로 딴 리드의 41%가 "연락 안 됨"으로 죽고 있다는 뜻. 유입 시간대(새벽·아침)와 업무시간(11:30~20:00)이 어긋나는 게 근본 원인.

⚠️ 1차 콜 후 방치 741건 = 부재의 68%

재콜 시스템이 사실상 없음. 한 번 부재나면 그대로 묻힘. 이 741건은 아직 살아있을 확률이 가장 높은 리드인데 방치 중.

⚠️ 실결제 159건(6.1%) vs 부재 1,082건

오더 83 + 결보 76 = 159건이 실제 성과. 부재로 묶인 1,082건에서 단 10%만 살려도 실결제 모수가 이론상 크게 늘 수 있음. 회수 여지가 성과 규모를 좌우.

⑥ 제안 — 팀을 갈아넣지 않고 시스템이 받치는 구조

1️⃣ 부재 리드 자동 알림톡 (계절 무관 안전망)

1차 부재 → 솔라피 알림톡 자동 발송. 누가 바쁜 날이든 리드가 안 죽는다. 팀 추가 공수 0. 과거 방치분 741건 + 앞으로 들어오는 부재까지 전부 커버.

2️⃣ 방학 전(~7/20) 오전콜 = 방문사원 집중

유입이 새벽·아침에 몰리는데, 방문이 저녁(16:30~)이라 방문사원 오전이 빈다. 이 시간에 아침 유입을 즉시 받아내는 구조. 방문사원 사업부 추가가 이 3주간 가장 값어치 있는 이유.

3️⃣ DB 자동 분배 (주먹구구 → 가용 인력 기준)

'저 여기부터 칠게요' 방식은 다 바쁜 날 구멍이 생긴다. DB 유입 시 그날 콜 가능한 인력에게만 자동 배정. 방문·교육·면접인 사람은 자동 제외.

🎯 핵심

팀이 신규 콜·방문이라는 고가치 일에 집중하고, 반복·방치되는 부분은 시스템이 자동으로 메운다. 지금도 잘하고 있으니, 안 새는 장치만 깔면 된다.

⑦ 알림톡은 이렇게 작동한다 — "2차 부재" 트리거

과거 741건을 한꺼번에 쏘지 않는다. 오래된 부재는 뜬금없어 오히려 역효과. 대신 사람이 1~2차 콜을 시도한 뒤, 그래도 못 닿은 리드만 자동으로 넘긴다. 상담원이 상태를 바꾸는 순간 알림톡이 나가는 구조.

1차 콜 → 부재 → 2차 콜 → 여전히 부재 → 상담원이 상태 변경 → 솔라피 알림톡 자동 발송 📩

상담원은 딱 하나만 — 2차 콜 후에도 부재면, 노션에서 체크박스 하나 누르거나 상태값 변경만 하면 됨. 나머지(발송·중복방지·기록)는 시스템이 처리.
